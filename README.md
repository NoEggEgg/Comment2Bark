# Comment2Bark

Typecho 评论 Bark 推送插件 - 将博客评论实时推送到 iOS 设备。

## 简介

[Bark](https://github.com/Finb/bark) 是一款 iOS 端的推送工具，Comment2Bark 插件可将博客评论实时推送到你的 iPhone/iPad，让你第一时间收到评论通知，无需频繁刷新后台。

## 功能特性

### 推送场景

| 场景          | 触发条件                 | 推送标题                      | 消息体                                |
| ------------- | ------------------------ | ----------------------------- | ------------------------------------- |
| 📩 新评论     | 访客发布新评论且审核通过 | 📩 【博客名】有新评论         | 「文章标题」\n👤 评论者：内容         |
| 💬 回复通知   | 访客回复已有评论         | 💬 {被回复者}的评论被回复     | 回复：「原评论摘要」\n👤 回复者：内容 |
| ⏳ 待审核     | 评论进入审核队列         | ⏳ 【博客名】有「待审」评论   | 「文章标题」\n👤 评论者：内容         |
| 🗑️ 垃圾评论 | 评论被标记为垃圾         | 🗑️ 【博客名】有「垃圾」评论 | 「文章标题」\n👤 评论者：内容         |
| ✅ 审核通过   | 后台手动审核通过         | ✅ 【博客名】评论审核已通过   | 「文章标题」\n👤 评论者：内容         |

### 核心能力

- 🔗 **精确链接** - 自动计算评论分页，点击推送直接跳转到对应评论位置
- 🔄 **自动重试** - 推送失败自动重试 3 次（间隔 1s、2s、4s）
- 🛡️ **安全处理** - 评论内容经过 HTML 过滤和 XSS 处理
- 📊 **调试模式** - 启用后可查看详细推送日志
- 🎨 **自定义配置** - 支持自定义图标、分组、提示音

## 安装

1. 下载插件压缩包
2. 解压到 Typecho 插件目录 `usr/plugins/`
3. 重命名文件夹为 `Comment2Bark`
4. 登录后台 → 「控制台」→「插件」
5. 启用插件，点击「设置」配置参数

## 配置

| 选项       | 说明                                                                            | 默认值                   |
| ---------- | ------------------------------------------------------------------------------- | ------------------------ |
| 服务器地址 | Bark 服务器 URL，私有部署时自定义                                               | `https://api.day.app/` |
| Bark Key   | 从 Bark App 中获取的唯一密钥（必填）                                            | -                        |
| 消息保存   | 是否在 Bark 中保存推送记录                                                      | 保存                     |
| 忽略自己   | 忽略博主本人的评论推送                                                          | 否                       |
| 博主 UID   | 博主用户 ID，用于判断是否本人评论                                               | 1                        |
| 推送图标   | 自定义推送图标 URL                                                              | -                        |
| 消息分组   | Bark 消息分组名称，便于分类管理                                                 | -                        |
| 提示音     | 推送提示音，可选[Bark 提示音列表](https://github.com/Finb/Bark/tree/master/Sounds) | -                        |
| 调试模式   | 开启后记录详细日志到错误日志                                                    | 禁用                     |

## 常见问题

**Q: 推送失败怎么办？**

- 检查 Bark Key 是否正确
- 确认服务器能访问 Bark 服务器
- 启用调试模式查看详细日志

**Q: 如何获取 Bark Key？**
打开 Bark App，系统自动生成密钥，格式为 `https://api.day.app/XXXXXXXX`。

**Q: 链接指向不对？**
插件会自动计算评论分页，支持按评论创建时间正序或倒序排列的博客。

## 环境要求

- Typecho 1.3.0+
- PHP 7.4+（推荐 8.0+）

## 技术实现

- **推送方式**：优先使用 cURL，fallback 到 file_get_contents
- **分页计算**：根据博客评论分页设置和排序方式，自动计算评论所在页码
- **链接构建**：使用 Typecho Widget_Archive 动态获取文章永久链接
- **错误处理**：完整的异常捕获和日志记录
