# Comment2Bark 插件

## 简介

Comment2Bark 是一个 Typecho 插件，用于将博客的评论、trackback 和 pingback 实时推送到 Bark 客户端，让博主能够及时收到评论通知。

## 功能特性

- ✅ 实时推送评论通知到 Bark 客户端
- ✅ 支持自定义 Bark 服务器地址
- ✅ 支持自定义推送图标、分组和提示音
- ✅ 支持设置是否忽略博主自己的评论
- ✅ 支持消息保存设置
- ✅ 支持调试模式，便于排查问题
- ✅ 支持 PHP 8.5+，解决了 curl_close() 弃用问题
- ✅ 自动检测并使用 cURL 或 file_get_contents 发送请求
- ✅ 完整的错误处理和日志记录
- ✅ 推送失败重试机制，提高推送可靠性
- ✅ 增强的评论内容安全处理，防止 XSS 攻击
- ✅ 优化的评论链接构建，支持分页评论
- ✅ 统一的推送参数格式，使用 Bark 最新 API 规范

## 安装方法

1. 下载插件压缩包
2. 将插件解压到 Typecho 插件目录 `/usr/plugins/` 下
3. 重命名文件夹为 `Comment2Bark`
4. 登录 Typecho 后台，进入「控制台 → 插件」
5. 找到「Comment2Bark」插件并点击「启用」
6. 点击「设置」按钮，配置插件参数

## 配置选项

| 配置项         | 说明                                                                                    | 默认值               |
| -------------- | --------------------------------------------------------------------------------------- | -------------------- |
| 服务器地址     | 自定义 Bark 服务器地址                                                                  | https://api.day.app/ |
| Bark Key       | 下载 Bark App 后获取的唯一密钥（必填）                                                  | -                    |
| 消息保存       | 设置是否在 Bark App 中保存推送消息                                                      | 保存                 |
| 忽略自己的评论 | 启用后，若评论者为博主，则不会向 Bark 发送通知                                          | 否                   |
| 博主 UID       | 自定义博主 UID                                                                          | 1                    |
| 推送图标       | 自定义 Bark 推送图标，图标需为完整的 URL 链接                                           | -                    |
| 消息分组       | 自定义 Bark 消息分组，用于消息分类                                                      | -                    |
| 提示音         | 自定义 Bark 消息提示音，参考[Bark Sounds](https://github.com/Finb/Bark/tree/master/Sounds) | -                    |
| 调试模式       | 启用后，会在日志中记录推送请求和响应信息                                                | 禁用                 |

## 使用方法

1. **下载 Bark App**：在 App Store 下载 Bark 应用
2. **获取 Bark Key**：打开 Bark App，复制生成的唯一密钥
3. **配置插件**：在 Typecho 后台配置插件，填写 Bark Key 和其他选项
4. **测试**：发表一条评论，检查是否收到 Bark 推送通知

## 推送效果

当有新评论时，Bark 客户端会收到如下格式的通知：

```
你的博客有了新评论

评论者名称 在「文章标题」中说：

评论内容

文章链接
```

## 技术实现

- **推送方式**：优先使用 cURL， fallback 到 file_get_contents
- **错误处理**：完整的异常捕获和错误处理机制
- **日志记录**：调试模式下记录详细的请求和响应信息
- **兼容性**：支持 PHP 7.2+，特别优化了 PHP 8.5+ 的兼容性
- **代码结构**：采用模块化设计，代码清晰易维护

## 常见问题

### 1. 推送失败怎么办？

- 检查 Bark Key 是否正确
- 检查服务器地址是否有效
- 启用调试模式，查看日志中的错误信息
- 确保服务器可以访问 Bark 服务器

### 2. 如何获取 Bark Key？

打开 Bark App，系统会自动生成一个唯一的 Bark Key，格式类似 `https://api.day.app/XXXXXXXX`，其中 `XXXXXXXX` 就是你的 Bark Key。

### 3. 如何自定义推送图标？

在配置面板中填写完整的图标 URL 链接，例如 `https://example.com/icon.png`。

### 4. 如何设置消息分组？

在配置面板中填写分组名称，例如 `博客评论`，这样所有评论通知都会归类到该分组下。
